## 📂 1. โครงสร้างไฟล์และจุดส่งมอบงาน (Deliverables)

ทุกไฟล์ถูกจัดระเบียบและวางไว้ใน Root โฟลเดอร์ของ GitHub เรียบร้อยแล้ว:

1.  **สถาปัตยกรรมระบบ (Part 0):**
    *   **ไฟล์:** `architecture.md`
    *   **รายละเอียด:** แผนภาพไดอะแกรมทางเดินข้อมูล (Excel ➡️ Raw ➡️ Staging ➡️ Trusted ➡️ Analytics) และการวิเคราะห์การออกแบบระบบ
2.  **เอกสารคุณลักษณะและข้อกำหนดข้อมูล (Part 1 & 3):**
    *   **ไฟล์:** `data_specification.md`
    *   **รายละเอียด:** ข้อมูลสถิติจริง (Profiling) ของชุดข้อมูลนักศึกษา 180 คน, ตารางอธิบายคอลัมน์ทั้งหมด (Data Dictionary 49 คอลัมน์) และตารางกำหนดสิทธิ์ความปลอดภัยข้อมูล (Column Access Matrix) ตามเกณฑ์ความปลอดภัยของโปรดักชัน
3.  **โค้ดท่อข้อมูลหลัก (Part 2):**
    *   **ไฟล์:** `pipeline.py`
    *   **รายละเอียด:** โค้ด ETL (Python + Pandas) ที่ทำหน้าที่ดึงข้อมูล แปลงประเภท ลบข้อมูลซ้ำ และโหลดเข้า DuckDB
4.  **สคริปต์ทดสอบและผลลัพธ์การรันซ้ำ (Part 2):**
    *   **ไฟล์:** `verify_idempotency.py`
    *   **รายละเอียด:** สคริปต์สำหรับรันจำลองการทำงาน 2 รอบติดต่อกัน เพื่อยืนยันว่าระบบของเรา **ไม่เกิดข้อมูลซ้ำซ้อน (Idempotency)** และระบบบันทึกประวัติการทำงาน (Audit Logs) บันทึกได้อย่างถูกต้อง
5.  **หน้าแดชบอร์ดวิเคราะห์และประวัติการรัน (Part 2):**
    *   **ไฟล์:** `dashboard.py`
    *   **รายละเอียด:** เว็บแอปพลิเคชันแดชบอร์ด (Streamlit) แสดงกราฟสถิตินักศึกษา และดึงข้อมูลประวัติการทำท่อข้อมูลออกมาพรีเซนต์
6.  **รายงานสรุปงานฉบับเต็มภาษาไทย:**
    *   **ไฟล์:** `SUBMISSION_REPORT.md`
    *   **รายละเอียด:** รวบรวมงานทั้งหมดเขียนสรุปเป็นภาษาไทยอย่างละเอียด เพื่อใช้สำหรับนำเสนออาจารย์ผู้ประเมิน

## ⚙️ 2. สิ่งที่เราได้ทำไปแล้วบ้าง? (What's Already Done?)

1.  **วิเคราะห์ข้อมูลจริง (Data Profiling):** รันสคริปต์สแกนข้อมูล Excel ทั้ง 180 แถวเพื่อหาผลรวมสะสมสำหรับตรวจคุณภาพ (เช่น ผลรวมหน่วยกิต = 11,599, ผลรวมเงินเดือน = 6,481,500 บาท)
2.  **พัฒนาท่อส่งข้อมูล (Data Pipeline):**
    *   **Raw Layer:** จัดเก็บไฟล์ Parquet ที่เหมือนไฟล์ต้นฉบับ 100% สำหรับรันประมวลผลย้อนหลัง
    *   **Staging Layer:** ปรับประเภทข้อมูลและคัดกรองแถวซ้ำ
    *   **Trusted Layer:** โหลดข้อมูลลงฐานข้อมูล DuckDB โดยเพิ่มกลยุทธ์ "ลบข้อมูลเก่าที่มี Snapshot Date เดียวกันออกก่อนบันทึกใหม่" ทำให้รันกี่รอบข้อมูลในตารางก็ไม่ซ้ำซ้อน
    *   **Analytics View:** สร้าง SQL View ในฐานข้อมูล DuckDB ชื่อ `analytics_student_summary` เพื่อดึงข้อมูลสถิตินักศึกษาแยกตามวิทยาเขตและสาขาโดยอัตโนมัติ
3.  **จัดระบบความปลอดภัย:** สร้าง Access Matrix เพื่อกำหนดนโยบายจัดเก็บข้อมูลส่วนบุคคล (PII)
4.  **ระบบบันทึกการทำงาน (Audit Logs):** สร้างตารางจัดเก็บข้อมูลการรันระบบโดยอัตโนมัติ (วันเวลาที่รัน, จำนวนแถว, สถานะ SUCCESS/FAILED)

## 💻 3. วิธีการนำโค้ดไปรันต่อบนเครื่องของทุกคน (How to run)

สำหรับสมาชิกในทีมคนอื่นๆ ที่โคลนโค้ดลงไปรันในเครื่องตัวเอง ให้ทำตามขั้นตอนดังนี้ครับ:

1.  **ติดตั้งไลบรารีที่จำเป็น:**
    ```bash
    pip install -r requirements.txt
    ```
2.  **รันระบบท่อข้อมูลและตรวจสอบ Idempotency อัตโนมัติ:**
    ```bash
    python3 verify_idempotency.py
    ```
3.  **รันหน้า Dashboard ขึ้นมาดู:**
    ```bash
    python3 -m streamlit run dashboard.py
    ```
